This module contains methods for working with a Git respository instance.

Behaviours: gen_server.

Description

This module contains methods for working with a Git respository instance.

The repository instance is created with the elibgit:open/1 function, and then can be used with subscript notation, like so: 
  {ok, Git} = elibgit:open(Path),
  {ok, Oid} = Git:get_ref("...").
Once you are finished with an instance, you should call the elibgit:close/1 function to close the port and clean up resources.

build_tree/4

elibgit:build_tree(StartingPoint::git_oid() | empty, Removes::[git_remove_op()], Inserts::[git_insert_op()], Inst::elibgit()) -> {ok, git_oid()} | {error, term()}

Builds a new tree using a basis and list of operations.

The new tree is initialised with identical contents to the StartingPoint tree. Then, the sub-trees and blobs specified in Removes are deleted, in order. Finally, the new sub-trees and blobs in Inserts are added in order, and the resulting tree written to the repository.

For example, to take the existing tree "abcd1234..." and add a new file, "bah.txt", you would first call create_blob/2 with the data to go inside it. Then, call build_tree with the StartingPoint as "abcd1234...", no Remove operations, and one Insert operation: {gitinsert, "bah.txt", "blob_oid_here", 0}.
